---

octo 的更新日志就是 GitHub Releases 页面——每个版本的发布说明由提交历史自动生成，按 Features /
Bug fixes 分组，任何影响 Stable 层级的改动都会带上 **Breaking** 标注
（见[兼容性](/docs/zh/reference/compatibility/)）：

**[github.com/open-octo/octo-agent/releases](https://github.com/open-octo/octo-agent/releases)**

Linux、macOS、Windows 的预编译二进制都挂在每个 release 下面。`octo upgrade`
会原地安装最新版；`octo upgrade --check` 只比较版本号。
