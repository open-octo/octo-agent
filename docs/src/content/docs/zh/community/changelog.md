octo 的更新日志放在仓库里——单个文件，按时间排列，任何影响 Stable 层级的改动都会带上
**Breaking** 标注（见[兼容性](/docs/zh/reference/compatibility/)）：

**[github.com/open-octo/octo-agent/blob/main/CHANGELOG.md](https://github.com/open-octo/octo-agent/blob/main/CHANGELOG.md)**

Linux、macOS、Windows 的 release 和预编译二进制都在
[GitHub Releases 页面](https://github.com/open-octo/octo-agent/releases)。`octo upgrade`
会原地安装最新版；`octo upgrade --check` 只比较版本号。
