---
title: 会话与历史
description: 持久化、恢复，以及崩溃时的可靠性。
---

每一次交互式对话都是一个会话，每一轮结束后都会以 JSONL 形式（一行一条记录，首行是 meta 头）持久化到 `~/.octo/sessions/` 下面。

```bash
octo sessions        # 列出已保存的会话
octo -c              # 从最近的列表里选一个恢复
octo -c <session-id> # 恢复指定的一个
```

## 崩溃时的可靠性

历史按"轮"为粒度持久化,所以中途崩溃最多丢掉正在进行的这一轮,而不是整个会话。一个回放缓冲区
覆盖了"最后一次持久化的轮次"和"进程死掉那一刻正在流式输出的内容"之间的空隙,所以恢复时能干净地
接上,而不是重放一段陈旧的状态。

## 消息格式

在内部,一条消息的内容是文本、tool-use、tool-result 这几种 block 的联合体,而不是单一字符串——
这正是恢复会话时能忠实还原工具调用的原因。这个机制出现之前保存的老会话依然能正常加载:
block 列表为空时会自动回退成纯字符串形式。

## 会话分支

在 Web UI 中,任意一条用户消息都可以分支: hover 消息点击 **Branch**,可以选择性地编辑 prompt,
然后创建一个新会话,历史复制到该消息(含)为止。原会话完全不动。这在调试 prompt 变体时很有用——
改写一个问题并排对比结果,不会污染原会话。

也可以**原地编辑**用户消息: hover 点击 **Edit**,消息变成输入框可以修改。保存后会截断该消息之后的历史
并重新发送修改后的 prompt,一切都在当前会话里完成——不新建会话。Branch 和 Edit 的唯一区别是是否新建会话。

新会话带有 `branched_from` 字段指向源会话,在会话标题旁显示"分支自 <标题>"。分支功能通过
`POST /api/sessions/{id}/branch` 提供,参数为 `message_index` 和可选的 `prompt_override`;
编辑功能通过 `POST /api/sessions/{id}/edit_message` 提供。
