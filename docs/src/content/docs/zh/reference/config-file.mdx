`~/.octo/config.yml` 保存你的默认 provider、model 和全局设置，这样裸跑 `octo` 就不用每次重敲
flag。所有字段都是可选的——文件不存在，或者字段缺失，都会回退到内置默认值。尽量用
`octo config` 管理它，而不是手改。

<Aside type="caution" title="迁移旧配置">
两种旧格式仍会被读取，并在下次 `octo config` 保存时升级。`~/.octo/config.yaml`（`.yml` 之前的
文件名）在 `config.yml` 不存在时会被读取，迁移后原文件保留为 `config.yaml.bak`。顶层扁平的
`models:` 列表（两级 schema 之前的写法）会被改写成 `endpoints:`——每个旧条目变成一个名为
`legacy-<host>-<n>` 的端点，你可以随意改名。
</Aside>

## 顶层字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `endpoints` | [端点](#端点) 列表 | 配置好的 provider，每个端点打包共享的连接参数和一组模型 |
| `default` | string | 复合 id `<端点id>::<model>`，没有其他选择时使用；留空则回退到第一个端点的第一个模型 |
| `lite` | string | 用于低成本内部调用（[历史压缩](/docs/zh/concepts/compaction/)的总结步骤和会话标题生成）的复合 id `<端点id>::<model>`；留空则从端点的 `lite_model` 或 vendor 默认推断 |
| `permission_mode` | string | `interactive` \| `strict` \| `auto` |
| `coauthor` | bool | 在 agent 写的 git commit 里追加 `Co-authored-by`（默认 `true`） |
| `access_key` | string | 暴露到回环之外时,Web UI / API 认证用的共享密钥 |
| `compact_auto_pct` | int | 自动压缩阈值,按上下文窗口的百分比（默认 `75`）；同一个阈值在轮次之间和每个工具调用批次后都会检查 |
| `reasoning_effort` | string | 全局推理强度：`low` \| `medium` \| `high` \| `xhigh` \| `max`；留空表示关闭 |
| `show_reasoning` | bool | 是否把推理轨迹提供给 Web UI 的全局默认值 |
| `workspace_dir` | string | 新建 Web 会话的默认工作目录；`"auto"` → `~/Desktop/octo` |
| `tools` | [tools 配置块](#tools-配置块) | 各工具的行为开关 |
| `browser` | [browser 配置块](#browser-配置块) | Chrome 连接设置 |
| `goal` | `{ enabled: bool }` | 控制 `/goal` 和目标相关工具（默认启用） |
| `trash` | [trash 配置块](#trash-配置块) | 文件回收站行为（覆盖备份、保留天数、容量上限） |
| `memory_backend` | object | 可选的外部语义记忆后端——见[记忆后端](/docs/zh/guides/memory-backends/) |
| `language` | string | UI 语言偏好（`en` \| `zh`）；留空为英文 |
| `notify` | bool | 仅 TUI：turn 完成等待输入时发送桌面通知。Ghostty / iTerm2 / Kitty 会转发到操作系统；其他终端回退到终端响铃（默认 `true`） |
| `terminal_title` | bool | 仅 TUI：启动时将终端标签/窗口标题设为会话名（OSC 2，Ghostty、iTerm2、Kitty 及大多数现代终端支持；默认 `true`） |

当 agent 编辑这个文件时,octo 会在写入后立刻校验:如果不再能解析、或有语义问题(`default` 指向不存在的端点、id 重复或字段不全),会在当前这一轮里给出警告——否则坏改动会静默降级,因为 octo 会继续用最后一份能正常解析的配置。

## 端点

`endpoints:` 里的每一项打包了一组模型共享的连接参数。两级结构让同一个 model 名字可以挂在
多个端点下——比如 `claude-sonnet-5` 同时走官方 Anthropic 端点和一个中转——这是旧的扁平列表
表达不了的。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一标识，也是复合 id 的前缀；匹配 `^[a-zA-Z0-9_-]+$`，不能含 `::`。向导按 provider 命名（`anthropic`、`openai`、`custom`），可随意改名 |
| `name` | string | 可选：UI 里显示的名字，可以和 `id` 不同 |
| `provider` | string | `anthropic` \| `openai` \| `custom` |
| `base_url` | string | 端点 URL；留空使用该 vendor 的默认端点。`custom` vendor 必填 |
| `api_key` | string | 明文兜底，仅当对应环境变量为空时使用（权限 `0600`） |
| `protocol` | string | 仅 `custom` vendor 使用的协议：`anthropic` \| `openai` |
| `lite_model` | string | 可选：指定本端点 `models` 里的某个模型作为它的压缩 lite 模型 |
| `models` | [模型](#模型) 列表 | 通过这个端点提供的模型 |

## 模型

端点 `models:` 列表里的每一项都可独立选择。`--model <name>` 会选中它——连同该端点的连接参数——
而 `default` / `lite` 通过复合 id `<端点id>::<model>` 引用它。连接设置（`base_url`、`api_key`、
`protocol`）在端点上，推理设置（`reasoning_effort`、`show_reasoning`）在顶层，都不放在这里。

| 字段 | 类型 | 说明 |
|---|---|---|
| `model` | string | 发给 API 的 model id（如 `claude-sonnet-4-6`）——也是 `--model` 和 HTTP API 用的名字 |
| `vision` | bool | 这个模型能否接收图片；模型级能力，对已知模型会自动判断 |

```yaml
endpoints:
  - id: anthropic
    provider: anthropic
    models:
      - model: claude-sonnet-5
        vision: true
  - id: deepseek
    provider: custom
    protocol: anthropic
    base_url: https://api.deepseek.com/anthropic
    models:
      - model: deepseek-chat
default: anthropic::claude-sonnet-5
lite: anthropic::claude-sonnet-5
reasoning_effort: ""      # low | medium | high | xhigh | max；留空 = 关闭
permission_mode: interactive
coauthor: true
```

## Tools 配置块

```yaml
tools:
  tool_search:
    enabled: auto          # auto（默认）| on | off
    threshold_pct: 10
  disabled_skills: []
```

`disabled_skills` 会把列出的 skill 从模型和 UI 里隐藏，但不会从磁盘删除。`tool_search`
的作用见[接入 MCP 服务](/docs/zh/guides/connect-mcp-servers/)。

## Browser 配置块

```yaml
browser:
  attach_running: true   # 复用你已登录的 Chrome，而不是用一个临时 profile
  connect_port: 9222      # 或者通过 --remote-debugging-port 接入
  user_data_dir: ""
  exec_path: ""
  download_dir: ""
```

详见[浏览器自动化](/docs/zh/guides/browser-automation/)。

## Trash 配置块

```yaml
trash:
  overwrite_backup: true  # write_file/edit_file 覆盖前先备份一份（git 已跟踪且干净的文件跳过）
  retention_days: 14      # 启动时清理超过天数的条目；负数关闭
  max_size_mb: 10240      # 超过容量上限时从最旧开始淘汰；负数关闭
```

回收站在安全模型里的位置见[沙箱化 agent](/docs/zh/guides/sandbox-the-agent/)；CLI 侧的恢复命令是
`octo trash list|restore|rm|empty`。

下一步：其中不少设置都有对应的 CLI flag 可做单次运行的临时覆盖——见 [CLI 参考](/docs/zh/reference/cli/)。
