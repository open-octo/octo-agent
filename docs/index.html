Meet Octo

A functionality-first AI agent in a single Go binary. Bring it to your terminal, browser, or chat app — it speaks Anthropic and OpenAI natively, and actually gets things done.

⭐ Star on GitHub 📖 Read the Docs
go install github.com/Leihb/octo-agent/cmd/octo@latest

Built for people who ship

Octo is not a thin wrapper around an API. It is a full agent loop with tools, memory, sandboxing, and autonomy — packaged as one file you can drop anywhere.

📦

Single binary, zero deps

One Go executable. No Python, no Node, no Docker. Download a prebuilt for macOS, Linux, or Windows and run it immediately.

🔌

Two protocols, every provider

Speaks Anthropic Messages and OpenAI Chat Completions natively. DeepSeek, Kimi, Bailian, OpenRouter, vLLM — if it exposes either shape, it just works.

🛠️

Real tools, not toy examples

File read/write/edit, terminal (with background jobs), glob, grep, web fetch/search. Plus MCP servers and reusable Skills in Claude Code format.

🧠

Persistent memory

Octo remembers your preferences, project conventions, and past corrections across sessions — stored locally in ~/.octo/memories/, not in the cloud.

🔒

OS-level sandboxing

Enable --sandbox to confine terminal commands to the project directory with no network access. macOS Seatbelt and Linux Landlock + seccomp.

🤖

Autonomous orchestration

octo conduct plans a goal into a subtask DAG, delegates to parallel sub-agents, and resumes after crashes. Long-horizon tasks without babysitting.

Three faces, one agent

Each interface is a first-class citizen. Use the one that fits your workflow — the agent underneath is identical.

💻

Terminal

Rich interactive TUI with streaming, tool-call cards, session history, and readline editing. Or run headless one-shots in scripts and CI.

🌐

Web UI

Local dashboard at :8080. Browse sessions, inspect tool output, manage skills — same agent, richer visuals.

💬

IM Bridge

WeChat iLink today. Scan a QR code, add Octo to your group chat, and delegate tasks from your phone without opening a terminal.

See it in action

A few commands to get a feel for what Octo can do.

One-shot agentic task
# Headless: one prompt → full tool loop → exit
octo "Add a --json flag to octo config show"

# Pipe-friendly for scripts
echo "Summarise the last commit" | octo
Interactive session
# TUI with history, tool cards, and /commands
octo
octo sessions
octo -c <session-id>

# With reasoning and sandbox
octo --reasoning-effort high --sandbox
Autonomous execution
# Plan, delegate, and run without babysitting
octo conduct "Refactor error handling in internal/agent"
octo conduct status <id>
octo conduct resume <id>
Configuration layers
# Octo learns who you are and how you work
~/.octo/soul.md      # agent persona
~/.octo/user.md      # your profile
~/.octo/octorules.md # global rules
.octorules           # project conventions

What Octo is

  • A tool that gets out of the way and lets you work
  • Functionality over token-counting minimalism
  • Open, inspectable, and hackable — MIT licensed
  • Local-first: your data stays on your machine
  • Composable: MCP, skills, hooks, sub-agents

What Octo is not

  • Not a web-only SaaS with a tacked-on CLI
  • Not a closed marketplace for encrypted skills
  • Not obsessed with shaving tokens at the cost of capability
  • Not a black box — every tool call is visible and gated
  • Not trying to replace your IDE (it pairs beautifully with one)
